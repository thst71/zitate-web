The idea is a mobile iOS application to maintain a datbase of sayings, citations and generally speaking "bon mots". The data should be recorded with a position on a map, with pictures, sound and of course the text itself. An author should be an optional addition, where the authors should not be duplicates. When we know the author, I want to keep some information on him, like a wikipedia page, personal coordinates, like date of birth, location, day of decease, etc. To maintain some order in the collection, the collectibles are tagged with labels and there is  a function to build intelligent folders based on any combination of features of the data. The combinations shall be formulated as near natural language sentences, we possibly want LLM support to create a logical expression from the sentence, so that searching is fast and LLM support is only needed to create the search expression not used during search.

The UI must adhere to the iOS platform requirements and it should have a cards layout on the front screen, where a simple search box and a text matching my location is selected. The data maintenance shall be with a master detail view for editing entries. 

Labels must be proposed from existing labels and be added on the fly. Case doesn't matter. Later I want an LLM (apple intelligence?) to suggest labels from my database and from the contents of the text collected. 

This should be in the version 1.0 of the application. Based on the description create a requirements file for me that breaks my ideas into SMART requirements. I want to review them and then create an architecture and implementation agents.md from it.
