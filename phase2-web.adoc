== Phase Overview

*Goal:* Author management and label/tag system

*Duration:* Weeks 3-4

== Stories in Phase 2

=== Epic 4: Author Management

==== Story 4.1: Author Data Model and CRUD
*As a* user +
*I want to* create and manage authors +
*So that* I can attribute quotes to their sources

*Acceptance Criteria:*
- Author object store in IndexedDB with name, createdAt, updatedAt
- Create, read, update, delete operations
- Author name validation (max 200 chars, required)
- Duplicate author prevention (case-insensitive)
- Authors sorted alphabetically

*Reference:* FR-2.1, FR-2.2

*Technical Tasks:*
1. Create `src/hooks/useAuthors.ts` custom hook
2. Implement addAuthor, updateAuthor, deleteAuthor, getAuthorById
3. Add duplicate detection logic
4. Sort authors alphabetically
5. Add unit tests for all CRUD operations
6. Test duplicate prevention

*Estimated Effort:* 5 hours

=== Epic 5: Label Management

==== Story 5.1: Label Data Model and CRUD
*As a* user +
*I want to* create and manage labels +
*So that* I can organize my entries

*Acceptance Criteria:*
- Label object store in IndexedDB with name, createdAt, updatedAt
- Create, read, delete operations (no update needed)
- Label name validation (max 50 chars, no comma/semicolon)
- Labels stored lowercase
- Duplicate label prevention
- Labels sorted alphabetically

*Reference:* FR-3.1, FR-3.3

*Technical Tasks:*
1. Create `src/hooks/useLabels.ts` custom hook
2. Implement addLabel, deleteLabel, getLabelsByIds
3. Add normalization (toLowerCase)
4. Add validation (no comma/semicolon)
5. Implement searchLabels for autocomplete
6. Add unit tests
7. Test duplicate handling

*Estimated Effort:* 5 hours

---

==== Story 5.2: Label Input Component with Autocomplete
*As a* user +
*I want to* add labels to my entry with autocomplete +
*So that* I can reuse existing labels easily

*Acceptance Criteria:*
- Text input for label entry
- Autocomplete dropdown shows matching labels (top 10)
- Prefix-based search (case-insensitive)
- Enter key or click adds label
- Labels displayed as visual tags below input
- Can remove labels by clicking X
- Creates new label if doesn't exist
- Returns existing label if matches (case-insensitive)

*Reference:* FR-3.2, FR-3.4

*Technical Tasks:*
1. Create `src/components/label/LabelInput.tsx`
2. Implement text input with onChange
3. Add autocomplete dropdown with searchLabels
4. Implement label tag display
5. Add remove label functionality
6. Style tags (rounded, colored, with X button)
7. Handle keyboard navigation
8. Add unit tests

*Estimated Effort:* 8 hours

## Risk Mitigation

=== Risk: Autocomplete Performance
*Mitigation:* Limit results to top 10, use prefix-based search, debounce input

=== Risk: Label Name Conflicts
*Mitigation:* Normalize to lowercase, prevent duplicates, return existing on match

=== Risk: Breaking Changes to Entry Model
*Mitigation:* Make authorId and labelIds optional, maintain backward compatibility
