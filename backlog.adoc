= Implementation Backlog: Zitate v1.0
:toc:

== Overview
This backlog contains user stories and implementation tasks derived from the requirements specification. Stories are organized by epic and prioritized for iterative development.

== Epic 1: Core Data Model and Persistence

=== Story 1.1: Entry Data Model
*As a* developer +
*I want to* create the core Entry data model with all required fields +
*So that* entries can be stored and retrieved from the database

*Acceptance Criteria:*

- Entry entity with fields: id, text, latitude, longitude, createdAt, updatedAt
- Text field with 1-10,000 character validation
- Location fields support nullable values
- Core Data schema created
- Database migration support implemented

*Reference:* FR-1.1, FR-1.2

---

=== Story 1.2: Author Data Model
*As a* developer +
*I want to* create the Author data model with metadata fields +
*So that* author information can be stored and linked to entries

*Acceptance Criteria:*

- Author entity with fields: id, name, dateOfBirth, locationOfBirth, dateOfDeath, wikipediaURL
- Name required, max 200 characters
- Optional fields properly nullable
- One-to-many relationship with Entry
- Unique constraint on name (case-insensitive)

*Reference:* FR-2.1, FR-2.2

---

=== Story 1.3: Label Data Model
*As a* developer +
*I want to* create the Label data model with many-to-many relationship to entries +
*So that* entries can be tagged with multiple labels

*Acceptance Criteria:*

- Label entity with fields: id, name
- Name stored in lowercase, max 50 characters
- Many-to-many relationship with Entry
- Unique constraint on name

*Reference:* FR-3.1

---

=== Story 1.4: Media Storage Model
*As a* developer +
*I want to* create models for storing image and audio file references +
*So that* media can be associated with entries

*Acceptance Criteria:*

- Image entity with fields: id, filePath, order, entryId
- Audio entity with fields: id, filePath, duration, entryId
- One-to-many relationship from Entry to Image (max 10)
- One-to-one relationship from Entry to Audio
- File path validation

*Reference:* FR-1.3, FR-1.4

---

=== Story 1.5: Smart Folder Data Model
*As a* developer +
*I want to* create the SmartFolder data model with criteria storage +
*So that* folder configurations can be persisted

*Acceptance Criteria:*

- SmartFolder entity with fields: id, name, criteriaJSON, order, createdAt
- Name required, max 100 characters
- Criteria stored as JSON string
- Order field for user-defined sorting

*Reference:* FR-4.2, FR-4.4

---

=== Story 1.6: iCloud Sync Configuration
*As a* developer +
*I want to* configure Core Data for optional iCloud sync +
*So that* users can backup their data to iCloud

*Acceptance Criteria:*

- Core Data stack configured for CloudKit
- iCloud capability added to project
- Sync option in app settings
- Conflict resolution strategy implemented

*Reference:* NFR-2

=== Story 4.4: Criteria Builder
*As a* user +
*I want to* build folder criteria using a structured form +
*So that* I can define my folder rules without using natural language

*Acceptance Criteria:*

- Labels section: multi-select with AND/OR toggle
- Author section: single-select dropdown
- Date range section: start/end date pickers
- Location section: map picker + radius slider (km)
- Text match section: text input field
- Preview showing match count
- All criteria optional (can combine any)

*Reference:* FR-4.2, FR-4.4

---

=== Story 4.5: Natural Language Query Processing
*As a* user +
*I want to* describe my folder criteria in natural language +
*So that* I can create folders without using a complex form

*Acceptance Criteria:*

- Text input field for natural language query
- Example placeholder: "quotes about love from 2024 near Paris"
- Submit button sends query to LLM
- Processing indicator shown
- LLM generates structured query (JSON)
- Preview shows matching entries
- Completes within 5 seconds
- Fallback to criteria builder if LLM unavailable (iOS 17.x)

*Reference:* FR-4.3, NFR-3

---

=== Story 4.6: Smart Folder Persistence
*As a* developer +
*I want to* store folder criteria as logical expressions +
*So that* folders update dynamically without re-querying the LLM

*Acceptance Criteria:*

- Criteria stored as JSON in database
- Query engine evaluates criteria against entries
- Folders update when entries change (add/edit/delete)
- No LLM required for subsequent folder updates
- Efficient database queries with indexes

*Reference:* FR-4.2, FR-4.3

== Definition of Done

A story is considered "Done" when:

1. Code is written and peer-reviewed
2. Unit tests written and passing
3. UI tests written for user-facing features
