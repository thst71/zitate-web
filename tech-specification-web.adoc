== Overview
This document defines the technical architecture, tools, frameworks, and dependencies for the Zitate web-based single-page application (SPA).

== Technology Stack

=== Primary Technologies
- *Platform:* Web (Desktop and Mobile Browsers)
- *Language:* JavaScript/TypeScript ES6+
- *UI Framework:* React 18+ or Vue 3+ or Vanilla JS
- *Build Tool:* Vite or Webpack 5
- *Package Manager:* npm or pnpm

=== Core Browser APIs
- *IndexedDB:* Local database and persistence
- *Service Worker:* Offline capability and PWA
- *Geolocation API:* GPS coordinates
- *MediaRecorder API:* Audio recording
- *MediaDevices API:* Camera access
- *File API:* Image upload
- *Notification API:* Optional notifications (future)

== Architecture Pattern

=== Component-Based Architecture
- *Components:* Reusable UI components (React/Vue/Web Components)
- *Hooks/Composables:* Shared logic and state management
- *Services:* Singleton business logic services
- *Models:* TypeScript interfaces for type safety

=== State Management
- *Local State:* Component state (useState/ref)
- *Global State:* Context API / Pinia / Zustand (lightweight)
- *Persistent State:* IndexedDB via custom hooks
- *Derived State:* Computed values from IndexedDB queries

=== Data Flow
```
User Interaction → Component → Hook → Service → IndexedDB
                    ↑                              ↓
                    └────── State Update ←─────────┘
```

=== Object Stores

==== entries
```typescript
interface Entry {
  id: string;                    // UUID v4
  text: string;                  // 1-10,000 chars
  latitude?: number;             // WGS84
  longitude?: number;            // WGS84
  authorId?: string;             // FK to authors
  labelIds: string[];            // FK array to labels
  imageIds: string[];            // FK array to images
  audioId?: string;              // FK to audio
  createdAt: number;             // Unix timestamp
  updatedAt: number;             // Unix timestamp
}

// Indexes: createdAt, latitude, longitude, text (fulltext)
```

==== authors
```typescript
interface Author {

  wikipediaURL?: string;         // URL string
}

// Indexes: name (unique)
```

==== labels

interface Label {

==== folders
```typescript
interface SmartFolder {
  id: string;                    // UUID v4
  name: string;                  // Max 100 chars
  criteria: FolderCriteria;      // Nested object
  order: number;                 // User-defined order
  createdAt: number;             // Unix timestamp
}

interface FolderCriteria {
  labels?: {
    values: string[];            // Label IDs
    operator: 'AND' | 'OR';
  };
  authorId?: string;
  dateRange?: {
    start?: number;              // Unix timestamp
    end?: number;                // Unix timestamp
  };
  location?: {
    latitude: number;
    longitude: number;
    radiusKm: number;
  };
  textMatch?: string;
}

// Indexes: order
```

== Performance Optimization

=== IndexedDB
- Indexes on frequently queried fields (createdAt, name, etc.)
- Cursor-based pagination for large result sets
- Batch operations for multiple inserts/updates
- Transaction management (readwrite vs readonly)

=== UI
- Virtual scrolling for lists >100 items (react-window, vue-virtual-scroller)
- Lazy loading images (IntersectionObserver)
- Debounced search (300ms)
- Code splitting by route (React.lazy, dynamic imports)
- Memoization for expensive computations (useMemo, computed)

=== Bundle Size
- Tree shaking (Vite/Webpack)
- Dynamic imports for heavy features (maps, image compression)
- Minification and compression (gzip/brotli)
- Target: <500KB initial bundle (gzipped)

=== Caching Strategy (Service Worker)
```javascript
// Cache-first for static assets
self.addEventListener('fetch', (event) => {
  if (isStaticAsset(event.request.url)) {
    event.respondWith(cacheFirst(event.request));
  } else {
    event.respondWith(networkFirst(event.request));
  }
});
```

== Security and Privacy

=== Data Protection
- All data stored locally in browser (IndexedDB)
- No server communication (no API calls)
- No cookies or tracking
- Content Security Policy (CSP) headers

=== Input Validation
- Client-side validation for all user inputs
- XSS prevention (sanitize HTML, use framework escaping)
- SQL injection N/A (no SQL database)

=== Permissions
- Geolocation: Requested with user-friendly prompt
- Camera: Requested on first image capture
- Microphone: Requested on first audio recording
- Storage: Automatic with quota management

== Performance Targets

=== Load Times
- Initial load: <2s (desktop), <4s (mobile)
- Time to Interactive: <3s (desktop), <5s (mobile)
- First Contentful Paint: <1.5s

=== Runtime Performance
- Search response: <300ms
- IndexedDB query: <100ms
- UI interactions: 60fps
- Virtual scroll: Smooth at 60fps

=== Resource Usage
- Memory: <100MB typical usage
- Storage: ~10MB per 100 entries (excluding media)
- IndexedDB: Monitor quota, warn at 80%

== Technical Risks and Mitigations

=== Risk: IndexedDB Quota Limits
- *Mitigation:* Monitor storage usage, provide export before limits reached, warn user

=== Risk: Browser Compatibility Issues
- *Mitigation:* Feature detection, graceful degradation, clear browser requirements

=== Risk: Data Loss
- *Mitigation:* Automatic export prompts, periodic backup reminders, import/export functionality

=== Risk: Performance with Large Datasets
- *Mitigation:* Virtual scrolling, pagination, IndexedDB cursors, lazy loading

=== Risk: Media Storage Limits
- *Mitigation:* Image compression, size limits, warn before adding large files
