== Stories in Phase 4

=== Epic 9: Entry Editing

==== Story 9.1: Edit Entry UI
*As a* user +
*I want to* edit an existing entry +
*So that* I can fix mistakes or update information

*Acceptance Criteria:*
- Edit button visible on entry cards (hover on desktop, always on mobile)
- Clicking opens modal with pre-filled EntryForm
- All fields editable: text, author, labels
- Save button updates entry
- Cancel discards changes
- Modal title shows "Edit Entry"

*Reference:* Implied by FR-1.5 (CRUD operations)

*Technical Tasks:*
1. Add edit button to EntryCard component
2. Update EntryForm to accept initialEntry prop
3. Pre-fill form fields when editing
4. Update Modal title prop for edit mode
5. Style edit button (pencil icon)
6. Handle edit state in App.tsx
7. Add unit tests for edit button
8. Test form pre-filling

*Estimated Effort:* 4 hours

## Performance Considerations

=== Edit Performance
- Minimal re-renders when editing
- Efficient change detection
- Form state isolated to modal

=== Delete Performance
- Immediate UI update (optimistic)
- Background IndexedDB deletion
- Batch deletes if bulk operation

=== Toast Performance
- Queue system for multiple toasts
- CSS transitions (not JS animations)
- Auto-cleanup of dismissed toasts

## Risk Mitigation

=== Risk: Data Loss on Edit
*Mitigation:* Keep original entry until save confirmed, validation prevents invalid saves

=== Risk: Accidental Deletion
*Mitigation:* Confirmation dialog with preview, clear destructive styling

=== Risk: Stale Data After Edit
*Mitigation:* Refresh entry list after update, optimistic updates with rollback on error

=== Risk: Complex Change Detection
*Mitigation:* Deep equality check with JSON.stringify, or shallow comparison of primitives
