## Performance Considerations

=== Upload Performance
- Compress images before storage
- Show progress for large images
- Limit to 10 images per entry
- Validate file size before compression

=== Display Performance
- Lazy load images on entry cards
- Create object URLs for thumbnails
- Revoke object URLs when done
- Limit thumbnail count (first 3)

=== Storage Considerations
- 2MB max per image (after compression)
- 20MB max per entry (10 × 2MB)
- IndexedDB quota handling
- Warn user if quota approaching

## Risk Mitigation

=== Risk: IndexedDB Quota Exceeded
*Mitigation:* Compress images aggressively, warn user, provide export/delete options

=== Risk: Large Image Upload Performance
*Mitigation:* Show progress indicator, compress in background, prevent multiple simultaneous uploads

=== Risk: Browser Compatibility (HEIC)
*Mitigation:* Convert HEIC to JPEG client-side, fallback to unsupported message

=== Risk: Memory Leaks from Object URLs
*Mitigation:* Always revoke object URLs in cleanup, use useEffect cleanup
